# Module 16: Escalating Late Work

**Duration:** ~20 mins | **Topics:** 1 | **Level:** Beginner

## Overview
Configure escalation actions when work exceeds SLA deadlines. Escalation ensures overdue work receives appropriate attention and management.

## Learning Objectives
- Configure escalation rules for SLA violations.
- Implement escalation actions and notifications.
- Monitor and respond to escalated work.

## Topic 1: Escalation Rules and Actions
### Overview
Escalation rules automatically trigger actions when cases exceed SLA deadlines. These actions can include notifications, priority changes, reassignments, or other interventions to address overdue work.

### Key Concepts
- **Escalation rule:** Triggers action on SLA violation
- **Escalation action:** What happens when SLA exceeded
- **Notifications:** Alert managers of overdue work
- **Reassignment:** Move to higher priority resources
- **Priority change:** Increase case importance
- **Automated actions:** System-triggered responses
- **Multiple escalations:** Different actions at different times

### How It Works
Developers configure escalation rules that specify what happens when cases exceed SLA deadlines. Actions might include sending notifications to managers, reassigning work to more senior resources, increasing case priority, or triggering automated processes. Multiple escalation rules can trigger at different times.

### Takeaways
- Escalation rules ensure overdue work gets attention
- Multiple action types handle different scenarios
- Notifications alert management of issues
- Reassignment routes to capable resources
- Automated escalations reduce manual intervention

## Module Summary
Escalation rules transform SLAs from passive monitoring to active management. By configuring appropriate escalation actions, organizations ensure that overdue work receives timely intervention and doesn't fall through the cracks.

## CSA Exam Tips
1. Understand that escalation rules trigger on SLA violations
2. Know various escalation actions: notifications, reassignments, priority changes
3. Remember that multiple escalations can trigger at different times
4. Understand the importance of escalation in SLA compliance
5. Be prepared to explain how escalations support proactive work management

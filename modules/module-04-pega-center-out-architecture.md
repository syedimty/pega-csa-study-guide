# Module 4: Pega Center-out Architecture

**Duration:** ~20 mins | **Topics:** 2 | **Level:** Beginner

## Overview
This module introduces the Pega Platform prescriptive, Center-out architecture. The Center-out approach emphasizes prescriptive design, reusable patterns, and out-of-the-box accessibility features that streamline creation of scalable, inclusive user experiences. It combines low-code configuration with built-in best practices and accessibility standards for compliance and usability.

## Learning Objectives
- Describe the use of the Center-out approach in Pega application development.
- Explain the benefits of using the Center-out architecture.
- Describe the use of patterns in a Center-out architecture.
- Explain how Center-out architecture ensures compliance with accessibility standards.
- Describe the accessibility features included in Pega Platform.
- Identify accessibility best practices during application development.

## Topic 1: Pega's Center-out Business Architecture
### Overview
Pega's Center-out approach to digital transformation puts the customer and strategic outcome at the center of application design. This approach enables organizations to transform workflows into streamlined, automated, and intelligent processes that improve operational efficiency, enhance customer experience, and boost employee productivity.

### Key Concepts
- **Center-out™:** Architecture approach putting customer and strategic outcome at the center of design
- **Traditional approach:** Layered architecture with siloed components
- **Prescriptive design:** Built-in best practices and design patterns
- **Reusable patterns:** Standardized solutions for common problems
- **Omnichannel support:** Consistent experiences across channels
- Digital transformation enables streamlined, automated processes
- Customer-centric design improves operational efficiency
- Intelligent automation enhances productivity

### How It Works
Center-out architecture contrasts with traditional software development that creates siloed components. Instead, Pega designs applications with the customer journey at the center, using prescriptive patterns and best practices to create consistent, automated, and intelligent workflows across all channels and touchpoints.

### Takeaways
- Center-out design centers customer and strategic outcomes in application architecture
- Prescriptive approach includes built-in best practices reducing development errors
- Patterns enable reuse and consistency across applications
- Low-code configuration supports rapid, consistent implementation
- Architecture supports omnichannel experiences naturally

## Topic 2: Accessibility in Center-out Development
### Overview
Center-out architecture ensures compliance with accessibility standards through built-in features and development best practices. Pega Platform includes accessibility features out-of-the-box, and provides guidance for developers to implement inclusive applications.

### Key Concepts
- **Built-in accessibility:** Features included in platform by default
- **Compliance standards:** Support for WCAG, ADA, and other accessibility guidelines
- **Inclusive design:** Applications usable by people with various abilities
- **Best practices:** Development patterns that ensure accessibility
- **Keyboard navigation:** Full application functionality via keyboard
- **Screen reader support:** Content properly structured for assistive technologies
- **Color contrast:** Readable text and interface elements for all users

### How It Works
Pega Platform provides accessibility features that developers use automatically through low-code configuration. Components are built with accessibility standards in mind, reducing the need for custom accessibility implementation. Developers follow best practices like proper semantic HTML, keyboard support, and color contrast to ensure inclusive experiences.

### Takeaways
- Center-out architecture includes accessibility by default, not as an afterthought
- Pega Platform components support accessibility standards out-of-the-box
- Low-code development reduces common accessibility mistakes
- Development best practices ensure compliance with WCAG and similar standards
- Inclusive design benefits all users, not just those with disabilities

## Module Summary
Pega's Center-out architecture represents a fundamental shift from traditional software development approaches. By putting the customer and strategic outcome at the center of application design, organizations can create more effective, efficient, and inclusive applications. The architecture provides prescriptive guidance, reusable patterns, and built-in best practices that streamline development while ensuring quality and compliance. Accessibility is built into the platform from the ground up, making inclusive design achievable without specialized expertise. This approach enables teams to deliver customer-centric digital transformations that improve operational efficiency and user satisfaction.

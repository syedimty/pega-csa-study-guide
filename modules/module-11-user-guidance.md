# Module 11: User Guidance

**Duration:** ~15 mins | **Topics:** 2 | **Level:** Beginner

## Overview
Guide users through the Case Life Cycle by identifying the current Case Status and adding instructions to enable users to complete assigned Tasks. This module focuses on helping users understand where they are in case processing and what actions they need to take.

## Learning Objectives
- Identify the role of Case Status in Case processing.
- Identify when Case Status changes occur during Case processing.
- Identify the role of instructions in assigning Tasks to users.
- Add instructions to an assigned Task.

## Topic 1: Case Status
### Overview
Case Status identifies where a case is in its lifecycle and communicates the current state to users. Status changes occur as cases progress through different stages and processes, providing users with clear visibility into case progress.

### Key Concepts
- **Case Status:** Current state indicator in case lifecycle
- **Status changes:** Occur at specific workflow transitions
- **User communication:** Informs users of case state
- **Progress visibility:** Shows where case is in process
- **Stage-based:** Usually aligned with lifecycle stages
- **Automatic updates:** Change based on workflow progression
- **Custom statuses:** Can be tailored to business needs

### How It Works
Case Status is automatically set as cases progress through different stages and processes in the Case Life Cycle. Status values like "In Progress," "Pending Approval," or "Completed" communicate the current state. Users can view status to understand where a case is and what's happening next.

### Takeaways
- Case Status provides progress visibility for users
- Changes automatically as cases progress through workflow
- Aligns with Case Life Cycle stages
- Can be customized for specific business needs
- Critical for user communication

## Topic 2: Assignment Instructions
### Overview
Assignment Instructions guide users through assigned Tasks by providing context, steps, and information needed to complete the work. Clear instructions reduce errors and improve efficiency.

### Key Concepts
- **Instructions:** Guidance for completing assigned tasks
- **Context:** Background information about the task
- **Step-by-step:** Break down work into clear steps
- **Data guidance:** Explain what data needs to be entered
- **References:** Link to documentation or policies
- **Clarity:** Written clearly for all user levels
- **Task completion:** Help users finish work correctly

### How It Works
Developers add instructions to Tasks in the workflow. Instructions can include context about why the task exists, step-by-step guidance for completion, explanations of required data, and references to relevant policies or documentation. Instructions appear to users when they receive the assignment.

### Takeaways
- Instructions guide users through task completion
- Provide context and step-by-step guidance
- Reduce errors by clarifying requirements
- Include references to relevant information
- Improve user efficiency and satisfaction

## Module Summary
User Guidance combines Case Status and Assignment Instructions to create a clear user experience throughout case processing. Case Status provides visibility into where cases are in their lifecycle, while Assignment Instructions ensure users understand what to do and how to do it. Together, these elements reduce confusion, improve task completion quality, and enhance overall user experience.

## CSA Exam Tips
1. Understand that Case Status shows current state in case lifecycle and changes automatically with workflow progression
2. Know that Assignment Instructions provide guidance to users on how to complete assigned tasks
3. Remember that both elements are critical for user communication and task completion
4. Understand that instructions can include context, steps, data guidance, and references
5. Be prepared to explain how clear status and instructions improve user experience

# Module 1: Low-code Defined

**Duration:** ~55 mins | **Topics:** 5 | **Level:** Beginner

## Overview
Pega Platform provides an application development environment where Citizen Developers and System Architects alike create digital solutions to achieve business outcomes. This module focuses on the fastest way to build applications using the main building blocks of Pega Platform in both App Studio and Dev Studio.

## Learning Objectives
- Describe the value of low-code application development.
- Recognize the purpose of each Pega Platform studio.
- Find and use the navigation elements in App Studio.
- Explain how Dev Studio differs from App Studio.
- Switch between App Studio and Dev Studio when developing in Pega Platform.

## Topic 1: Low-code Application Development
### Overview
Low-code application development uses visual, declarative techniques instead of traditional programming. Pega has been pioneering this approach for 30 years, creating "Software That Writes Your Software."

### Key Concepts
- Forrester Definition: Products and/or cloud services for application development that employ visual, declarative techniques instead of programming
- Low-code tools create code automatically as developers manipulate and expand the visual model
- Drag-and-drop, process flows, and visual tools enable anyone to create transformational software regardless of technical ability
- Omnichannel capabilities: web pages, mobile applications, chatbots, email integration, and robotic data extraction

### How It Works
Users manipulate visual models and the low-code development tools automatically generate the underlying code. User interface features like drag-and-drop and process flows allow developers to focus on business logic rather than complex coding. The platform supports multiple channels (web, mobile, chatbot, email) through a unified visual development approach.

### Takeaways
- Low-code development increases productivity and supports rapid customization by streamlining app development with visual tools
- Pega enables creation of robust applications across multiple channels from a single visual model
- Integration with external systems is simplified through visual interfaces for data mapping

## Topic 2: Studios
### Overview
Pega Platform provides four role-based authoring spaces (studios), each with specific tools and features designed for different team members and expertise levels.

### Key Concepts
- **App Studio:** Low-code authoring space for business architects, citizen developers, and system architects with core features for case design, data management, and user experience
- **Dev Studio:** Advanced configuration authoring space for technical team members and system architects
- **Prediction Studio:** Analytics space for data scientists and business decision-makers
- **Admin Studio:** System administration space managed by IT staff
- Studio navigation consists of three areas: header, navigation pane, and authoring space
- Navigation pane includes: Overview (Case Types and Channels), Data, Library (Reuse Library), and Users

### How It Works
Each user has access to one or more studios and opens their default studio upon login. Users can switch between studios they have security access to. As a best practice, system architects work from App Studio unless required features are unavailable there, as App Studio manages behind-the-scenes configuration to reduce programming errors.

### Takeaways
- Each studio provides role-based functionality suited to different team members
- App Studio is the recommended starting point even for technical teams
- Studios enable improved productivity through specialized tools aligned with user expertise

## Topic 3: App Studio
### Overview
App Studio provides core features for application development including case design, data management, and user experience optimization. It is designed for low-code users including application developers, front-end developers, data engineers, business analysts, and system architects.

### Key Concepts
- App Studio visualizes key factors of desired business processes
- Helps plan and manage development team workloads
- Builds relationships between process stages, participating Personas, communication channels, and required data
- Visual model supports process planning and team management
- Features support agile development with rapid iteration capabilities

### How It Works
In App Studio, developers outline business processes visually and build relationships between stages of the process, participating Personas, communication channels, and data required for process resolution. The visual model helps teams plan and manage their development workload effectively.

### Takeaways
- App Studio provides visual tools for comprehensive process modeling
- The platform supports collaboration by creating a common visual language
- Development team workload management is built into the visual planning interface

## Topic 4: Dev Studio
### Overview
Dev Studio is a low-code authoring space with advanced configuration options for technical team members and system architects who need deeper customization capabilities.

### Key Concepts
- Provides advanced configuration options beyond App Studio's capabilities
- Designed for technical team members and system architects
- Supports deeper technical customization while maintaining low-code principles
- Menu options adapt based on screen size (text labels or icons)

### How It Works
Dev Studio serves as the advanced development environment when App Studio features are insufficient for specific requirements. It provides more granular control over application configuration while still leveraging Pega's low-code approach.

### Takeaways
- Dev Studio extends beyond App Studio for technical customization
- Platform supports seamless switching between App and Dev Studios based on feature requirements
- Both studios maintain the low-code philosophy while offering different complexity levels

## Topic 5: Integrating with External Applications
### Overview
Pega Platform simplifies integration with external systems and data sources through visual interfaces for configuration and data mapping.

### Key Concepts
- External data integration through visual interfaces
- Graphical data mapping capabilities
- Support for extracting data from multiple sources (emails, other software solutions)
- Enables applications to access data necessary to drive business processes forward

### How It Works
Through the visual interface, Pega Platform empowers users to build integrations to external systems to obtain necessary data. The platform provides graphical mapping of data into applications, offering a holistic view of the data landscape and enabling seamless process execution.

### Takeaways
- Integration capabilities are accessible through visual interfaces, not requiring code
- External data sources are essential components of robust application solutions
- Platform supports comprehensive data mapping and access patterns

## Module Summary
This module establishes the foundation for understanding Pega Platform's low-code approach to application development. It introduces the role of studios in supporting different team members, with App Studio as the primary development environment and Dev Studio for advanced customization. The module emphasizes that Pega's philosophy of "Software That Writes Your Software" enables teams of any technical level to create robust, multi-channel applications. Integration capabilities and visual modeling approaches enable developers to focus on business logic while the platform handles technical implementation. Understanding these core concepts is essential for effective application development in Pega Platform.

## CSA Exam Tips
1. Remember that App Studio is the recommended starting point even for technical teams because it manages behind-the-scenes configuration and reduces programming errors
2. Understand the four studios (App, Dev, Prediction, Admin) and their primary purposes - you'll likely see questions on which studio to use for specific tasks
3. Know that low-code doesn't mean "no code" - it means visual, declarative techniques replace traditional programming approaches
4. Understand the value proposition of low-code: increased productivity, rapid customization, agile development, and stakeholder collaboration
5. Be prepared to explain how omnichannel development in Pega allows creation of multiple channel applications from a single visual model

# Module 31: Creating a Child Case

**Duration:** ~25 mins | **Topics:** 1 | **Level:** Intermediate

## Overview
Implement logic to create child cases that support complex business processes. Child cases enable decomposition of work into manageable pieces while maintaining relationships.

## Learning Objectives
- Design child case creation logic.
- Configure child case relationships.
- Implement case hierarchy patterns.

## Topic 1: Child Case Creation and Management
### Overview
Child cases break complex work into related sub-cases, each with its own lifecycle while remaining linked to the parent case.

### Key Concepts
- **Parent case:** Main case holding related work
- **Child case:** Sub-case supporting parent
- **Case hierarchy:** Parent-child relationships
- **Creation trigger:** When child cases are created
- **Data inheritance:** Child inherits from parent
- **Relationship:** Link between parent and child
- **Completion logic:** Handle child case completion
- **Rollup:** Aggregate child status to parent

### How It Works
Applications create child cases when needed, inheriting relevant data from the parent case. Child cases proceed through their own lifecycle while remaining linked to the parent. Parent cases can track child completion and aggregate results.

### Takeaways
- Child cases decompose complex work
- Maintain parent-child relationships
- Child cases have independent lifecycles
- Data inheritance reduces duplication
- Enables management of work hierarchies

## Module Summary
Child cases enable sophisticated case hierarchies that decompose complex work into manageable pieces. By implementing appropriate child case logic, organizations handle intricate processes effectively.

## CSA Exam Tips
1. Understand parent-child case relationships
2. Know that child cases inherit relevant data from parent
3. Remember that child cases have independent lifecycles
4. Understand hierarchy management and completion tracking
5. Be prepared to explain when child cases are appropriate
